How to use

You can use MedSpaNER in two ways:

  • Via command-line executable code; instructions are included in this repository.

  • Via the graphical user interface (see image):

    A text is copied on the text box (A) and several annotation options can be selected (B).

    The annotated entities are shown with different colors (C).

    Hovering over an entity shows information about its type and normalization (e.g. UMLS concept unique identifiers), if available (D).

    Results can be downloaded in BRAT format (E), by clicking on the Download button (E).

Annotation options

By selecting the corresponding box, the system allows you to identify the following type of information:

   UMLS entities  

  • ANAT: anatomy or body parts: e.g., brazo ('arm')

  • CHEM: chemical or pharmacological sustances: e.g., aspirina ('aspirin')

  • DISO: pathologic conditions: e.g., diabetes

  • DEVI: medical devices: e.g., muletas ('crutches')

  • LIVB: living beings, included the human being: e.g., mouse ('ratón')

  • PHYS: physiological processes: e.g., digestión ('digestion')

  • PROC: therapeutic or diagnostic procedures and laboratory analysis: e.g., endoscopy ('endoscopia')

   Medical drug information  

  • Contraindicated: a contraindicated drug or treatment: e.g., contraindicación a aspirina ('aspirin')

  • Dose: dose or strength: e.g., 1 mg

  • Form: dosage form: e.g., pastillas ('pills')

  • Route: administration route or mode: e.g., vía oral ('oral route')

   Temporal expressions  

  • Age: age or life periods: e.g., 18 años ('18 years old')

  • Date: e.g., 2020

  • Duration: e.g., 2 horas ('2 hours')

  • Frequency: e.g., mensualmente ('monthly')

  • Time: hours or parts of the day: e.g., por la noche ('at night')

   Miscellaneous medical entities   

  • Concept: abstract concepts, statistical tests or measurement scales: e.g., ANOVA

  • Food: foods or drinks: e.g., alcohol

  • Observation: medical observations or clinical findings: e.g., normotenso ('normal blood pressure')

  • Quantifier_or_Qualifier: quantifier or qualifier adjective: e.g., grave ('severe')

  • Result_or_Value: result or value of a measurement, laboratory analysis or procedure: e.g., >90 mmHg

   Negation/Speculation  

  • Neg_cue: negation cue: e.g., no, sin ('no', 'without')

  • Negated: negated event: e.g., sin fiebre ('no fever')

  • Spec_cue: speculation cue: e.g., sospecha de ('suspicion of')

  • Speculated: speculated or uncertain event: e.g., posible infección ('possible infection')

   Attributes  

  • Temporality:

    • History_of: past event: e.g., antecedentes de migraña ('history of migraine')

    • Future: future event: e.g., cirugía pendiente ('pending surgery')

  • Experiencer:

    • Patient: patient or participant on a clinical trial: e.g., mujer embarazada ('pregnant woman')

    • Family_member: e.g., padre ('father')

    • Other: other person different from the patient or the family member: e.g., cirujano ('surgeon')

Use lexicon

If you select this option, the tool will annotate the text using the MedLexSp medical lexicon for natural language processing in Spanish.

The current version of MedLexSp contains 100887 lemmas and 302543 variant forms, which have been mapped to 42958 concept unique identifiers (CUIs) from the Unified Medical Language System (UMLS) of the National Library of Medicine (USA).

You can obtain the MedLexSp lexicon by signing the usage license available at Digital CSIC.

The recall (or sensibility) obtained with the lexicon is often higher, but its precision (positive predictive value) is lower than that obtained with the neural models.

💡 We recommend using the lexicon in pre-annotation tasks in which professionals can review the annotated data prior to use.

Nested entities

If this option is selected, the tool annotates entities that may appear within other entities with a wider scope.

For example, postoperatorio ('post-operatory', Date) in dolor postoperatorio ('post-operatory pain', DISO).

Nested entities are only shown if they belong to different types.

For example, in the entity dedo del pie ('toe'), the tool only annotates the full mention and not dedo, since they both have the same semantic class (ANAT).

By default, the tool retrieves only entities with a wider scope.

Nested entities can only be annotated if you use different models corresponding each to a group of entities (i.e. each selected option). For example, if both the UMLS entities and the Temporal Expressions options are selected. The external entity corresponds to one selected option and the internal entity corresponds to the other option.

Option Use lexicon does allow to annotate a wider range of nested entities.

Normalize

If you check this option, the system will map the annotated medical entities to the concept unique identifiers (CUIs) from the Unified Medical Language System (UMLS), which is maintained by the National Library of Medicine (USA).

For example, the entity cáncer de pulmón no microcítico ('non-small-cell lung cancer') has the CUI code C0007131 in the UMLS (see previous image).

An entity may have one or more unique concept identifiers, or none at all.

The command-line version of MedSpaNER also allows normalizing entities according to the Systematized Nomenclature of Medicine – Clinical Terms (SNOMED-CT) standard.

⚠️ Please note that the normalization of concepts has still not been evaluated and needs further review by professionals.

How it works

The tool has been developed using manually annotated texts about clinical trials, which were revised by experts. The texts were used to train neural models for medical entity recognition.

Texts were annotated according to the annotation criteria recorded in the annotation guidelines available.

The annotated texts (CT-EBM-SP vs 2 corpus) are available in this repository.

The default neural models used in this tool were fine-tuned using the RoBERTA bsc-bio-ehr-es base model; this model was trained by the Barcelona Supercomputing Center using clinical data and medical reports. The model was fine-tuned with annotated texts of clinical trials for different tasks:

  • recognition of medical entities from the UMLS semantic groups

  • annotation of temporal expressions

  • medical drug information

  • negation/speculation

  • attributes related to the temporality of the event or the experiencer

  • miscellaneous medical entities

All trained models are available for download at the Hugging Face repository.
